## 🚨 Non-Interactive Commands (CRITICAL)

**ALL terminal commands MUST be non-interactive.** Commands that wait for user input will hang indefinitely.

| ❌ NEVER USE | ✅ ALWAYS USE | Why |
|--------------|---------------|-----|
| `npx playwright test` | `npx playwright test --reporter=list` | Default may open interactive report |
| `npm test` (alone) | `npm run test:smoke` or specific script | May enter watch mode |
| `npx playwright test --ui` | `npx playwright test --reporter=list` | UI mode is interactive |
| `npx playwright show-report` | Read HTML file or use `--reporter=list` | Opens browser interactively |

### Output Capture Pattern

```powershell
# ✅ CORRECT - Capture output to file first, then filter
npx playwright test --reporter=list 2>&1 | Out-File -FilePath "playwright-output.txt" -Encoding utf8
Get-Content "playwright-output.txt" | Select-String -Pattern "passed|failed" | Select-Object -Last 30

# ❌ WRONG - Direct pipe loses output on filter miss
npx playwright test --reporter=list 2>&1 | Select-String -Pattern "passed"
```

## Test Categories

All tests are tagged with category markers. Tags are applied in `test.describe()` names.

### Priority Tags

- `@p0` - Critical path (must pass before release)
- `@p1` - Core features (should pass for stable release)  
- `@p2` - Extended coverage (nice to have)

### Feature Tags

- `@auth` - Authentication flows
- `@album` - Album CRUD operations
- `@photo` - Photo upload/view/download/delete
- `@sharing` - Share links and collaboration
- `@sync` - Data synchronization
- `@gallery` - Gallery view and navigation
- `@security` - Security and error handling
- `@a11y` - Accessibility compliance
- `@ui` - UI interactions

### Speed Tags

- `@fast` - Quick tests (<10s)
- `@slow` - Slow tests (>30s)

### Special Tags

- `@smoke` - Minimal verification set
- `@critical` - End-to-end user journeys
- `@multi-user` - Requires multiple browser contexts
- `@crypto` - Involves encryption/decryption

## Running Categories

```bash
# Single category
npx playwright test --grep @smoke
npx playwright test --grep @auth

# Multiple categories (OR)
npx playwright test --grep "@p0|@p1"

# Exclude category
npx playwright test --grep-invert @slow

# Combine with browser
npx playwright test --grep @smoke --project=chromium
```

## When Adding New Tests

1. **Choose appropriate priority tag** (`@p0`, `@p1`, or `@p2`)
2. **Add relevant feature tags** (`@auth`, `@photo`, etc.)
3. **Add speed tag if applicable** (`@fast` or `@slow`)
4. **Add special tags as needed** (`@smoke`, `@multi-user`, `@crypto`)

Example for a new sharing test:
```typescript
test.describe('New Sharing Feature @p1 @sharing @multi-user', () => {
  test('owner can revoke access @smoke', async ({ page }) => {
```

## CI Recommendations

- **Pre-commit:** `npm run test:smoke` (~2 minutes)
- **PR checks:** `npm run test:ci` (P0 + P1, ~10 minutes)
- **Nightly:** `npm test` (all tests, ~30 minutes)
- **Security audit:** `npm run test:security`
