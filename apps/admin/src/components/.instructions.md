## Component Patterns

### File Naming
- `PascalCase.tsx` for components (e.g., `PhotoGrid.tsx`)
- One component per file (unless tightly coupled)
- Index files for directory exports: `index.ts`

### Component Structure

```tsx
// 1. Imports (React, libraries, local)
import { useState, useCallback } from 'react';
import { usePhotos } from '../../hooks/usePhotos';
import type { Photo } from '../../types';

// 2. Types (props, local state)
interface PhotoCardProps {
  photo: Photo;
  onSelect?: (id: string) => void;
  isSelected?: boolean;
}

// 3. Component (exported, named function)
export function PhotoCard({ photo, onSelect, isSelected = false }: PhotoCardProps) {
  // 3a. Hooks first
  const [isLoading, setIsLoading] = useState(false);
  
  // 3b. Derived state
  const thumbnailUrl = photo.thumbnailUrl ?? '/placeholder.png';
  
  // 3c. Callbacks (memoized if passed to children)
  const handleClick = useCallback(() => {
    onSelect?.(photo.id);
  }, [photo.id, onSelect]);
  
  // 3d. Effects (minimal, prefer hooks)
  
  // 3e. Render
  return (
    <div onClick={handleClick} data-selected={isSelected}>
      <img src={thumbnailUrl} alt={photo.name} />
    </div>
  );
}
```

### Styling Approach

- **Tailwind CSS** for utility-first styling
- **CSS Modules** for complex animations (in `src/styles/`)
- **CSS Variables** for theme values
- NO inline style objects unless dynamic

```tsx
// âś… Good: Tailwind utilities
<button className="px-4 py-2 bg-blue-600 hover:bg-blue-700 rounded-lg">

// âś… Good: Dynamic with conditional classes
<div className={`p-4 ${isSelected ? 'ring-2 ring-blue-500' : ''}`}>

// âťŚ Bad: Inline styles
<button style={{ padding: '16px', backgroundColor: 'blue' }}>
```

## Component Categories

### Container Components
- Located in feature directories (`Albums/`, `Gallery/`)
- Manage state via hooks
- Handle data fetching and mutations
- Minimal styling, delegate to presentational components

### Presentational Components
- Located in `Shared/` or alongside containers
- Receive data via props
- Pure rendering, no side effects
- Reusable across features

### Page Components
- Top-level route components
- Compose containers and handle routing params
- Wrap with error boundaries

## State Management Rules

### Use Hooks for:
- Server state: `useAlbums()`, `usePhotos()`
- Feature logic: `useSelection()`, `useLightbox()`
- Session state: `useSession()`

### Use Context for:
- Cross-cutting concerns: `UploadContext`, `SyncContext`
- Feature permissions: `AlbumPermissionsContext`

### Use Local State for:
- UI state (open/closed, hover, focus)
- Form inputs before submission
- Animation state

```tsx
// âś… Good: Hook for feature logic
const { selectedIds, toggleSelection, clearSelection } = useSelection();

// âś… Good: Context for cross-cutting concern
const { uploadProgress } = useUploadContext();

// âś… Good: Local state for UI
const [isDropdownOpen, setIsDropdownOpen] = useState(false);
```

## Accessibility Requirements

- All interactive elements need `role` or semantic HTML
- Images need `alt` text (use empty string for decorative)
- Keyboard navigation for all actions
- Focus management in modals/overlays

```tsx
// âś… Good: Semantic HTML with keyboard support
<button onClick={handleDelete} aria-label="Delete photo">
  <TrashIcon />
</button>

// âś… Good: ARIA for custom controls
<div 
  role="checkbox" 
  aria-checked={isSelected}
  tabIndex={0}
  onKeyDown={(e) => e.key === 'Enter' && toggleSelection()}
>
```

## Error Boundaries

Wrap feature areas with error boundaries:

```tsx
import { ErrorBoundary } from '../ErrorBoundary';

export function GalleryPage() {
  return (
    <ErrorBoundary fallback={<GalleryError />}>
      <PhotoGrid albumId={albumId} />
    </ErrorBoundary>
  );
}
```

## Testing Components

- Co-locate tests: `ComponentName.test.tsx`
- Test user interactions, not implementation
- Mock hooks, not internal state
- Use `@testing-library/react`

```tsx
// âś… Good: Test user behavior
it('should select photo when clicked', async () => {
  const onSelect = vi.fn();
  render(<PhotoCard photo={mockPhoto} onSelect={onSelect} />);
  
  await userEvent.click(screen.getByRole('img'));
  
  expect(onSelect).toHaveBeenCalledWith(mockPhoto.id);
});
```

## Performance Patterns

### Virtualization
Use `@tanstack/react-virtual` for lists > 100 items:

```tsx
const virtualizer = useVirtualizer({
  count: photos.length,
  getScrollElement: () => containerRef.current,
  estimateSize: () => 200,
});
```

### Lazy Loading
Use `React.lazy` for route-level code splitting:

```tsx
const MapView = lazy(() => import('./MapView'));
```

### Memoization
Memoize expensive computations and stable callbacks:

```tsx
const sortedPhotos = useMemo(
  () => photos.sort((a, b) => b.createdAt - a.createdAt),
  [photos]
);
```
