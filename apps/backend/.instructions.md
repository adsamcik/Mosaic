## 🚨 Non-Interactive Commands (CRITICAL)

**ALL terminal commands MUST be non-interactive.**

| Task | ❌ NEVER USE | ✅ ALWAYS USE |
|------|--------------|---------------|
| Run tests | — | `dotnet test` |
| Build | — | `dotnet build` |
| Run server | `dotnet watch run` | `dotnet run` with `isBackground=true` |
| Restore packages | — | `dotnet restore` |
| Clean build | — | `dotnet clean` |
| Publish release | — | `dotnet publish -c Release` |
| Add package | — | `dotnet add package <name>` |
| Add migration | — | `dotnet ef migrations add <name>` |
| Update database | — | `dotnet ef database update` |
| Remove migration | — | `dotnet ef migrations remove` |
| List migrations | — | `dotnet ef migrations list` |
| Format code | — | `dotnet format` |

## Technology Stack

- **Framework:** .NET 10 with ASP.NET Core
- **Database:** PostgreSQL with Entity Framework Core
- **Upload:** Tus protocol for resumable uploads
- **Auth:** Trusted reverse proxy (Remote-User header) or Local auth with Ed25519 challenge

## Strict Constraints

### Zero-Knowledge Architecture
- The backend NEVER inspects, parses, or processes encrypted content
- All user content (photos, metadata) is treated as opaque `byte[]`
- The server cannot and should not decrypt anything

### Code Style
- Use `PascalCase` for all file names
- Use `async/await` consistently—never `.Result` or `.Wait()`
- Use `TypedResults` for compile-time verified responses in minimal APIs
- Prefer Controllers over minimal APIs for complex endpoints

### Error Handling
```csharp
// ✅ Good: Specific exception types
catch (EntityNotFoundException ex)
{
    _logger.LogWarning("Entity not found: {Id}", ex.EntityId);
    return NotFound();
}

// ❌ Bad: Generic exception swallowing
catch (Exception)
{
    return BadRequest();
}
```

### Database Patterns
```csharp
// ✅ Good: Split queries for multiple includes
await _db.Albums
    .Include(a => a.Members)
    .Include(a => a.Photos)
    .AsSplitQuery()
    .FirstOrDefaultAsync(a => a.Id == id);

// ✅ Good: Transactions for multi-step operations
await using var tx = await _db.Database.BeginTransactionAsync();
try
{
    // operations
    await tx.CommitAsync();
}
catch
{
    await tx.RollbackAsync();
    throw;
}

// ❌ Bad: N+1 queries
foreach (var album in albums)
{
    var photos = await _db.Photos.Where(p => p.AlbumId == album.Id).ToListAsync();
}
```

### Validation
- Input validation belongs in the Controller/Endpoint
- Use Data Annotations or FluentValidation
- Validate before any database operation

### Middleware Order (Critical)
```csharp
// Correct order in Program.cs
app.UseMiddleware<GlobalExceptionMiddleware>();  // 1. Catch all errors first
app.UseMiddleware<CorrelationIdMiddleware>();     // 2. Generate/extract correlation ID
app.UseLogScope();                                 // 3. Create logging scope
app.UseMiddleware<RequestTimingMiddleware>();     // 4. Log request timing
app.UseMiddleware<CombinedAuthMiddleware>();      // 5. Auth (supports LocalAuth + ProxyAuth)
app.UseAuthentication();                           // 6. ASP.NET Core auth
app.UseAuthorization();                            // 7. ASP.NET Core authz
app.UseAdminAuth();                                // 8. Admin endpoint protection
```

**Available Middleware:**
- `GlobalExceptionMiddleware` - Unified error handling
- `CorrelationIdMiddleware` - Request tracing
- `LogScopeMiddleware` - Structured logging context
- `RequestTimingMiddleware` - Performance logging
- `CombinedAuthMiddleware` - Handles both LocalAuth and ProxyAuth
- `TrustedProxyMiddleware` - Validates proxy headers
- `LocalAuthMiddleware` - LocalAuth-specific logic
- `AdminAuthMiddleware` - Protects /api/admin endpoints

## Testing Requirements

- Use xUnit with `WebApplicationFactory<Program>`
- Use in-memory SQLite for unit tests (with PostgreSQL-specific tests marked `[Trait("Database", "PostgreSQL")]`)
- Test both success and error paths
- Mock external dependencies, not the database

```csharp
public class AlbumsControllerTests : IClassFixture<WebApplicationFactory<Program>>
{
    [Fact]
    public async Task GetAlbum_WhenNotFound_Returns404()
    {
        // Arrange
        var client = _factory.CreateClient();
        
        // Act
        var response = await client.GetAsync($"/api/albums/{Guid.NewGuid()}");
        
        // Assert
        Assert.Equal(HttpStatusCode.NotFound, response.StatusCode);
    }
}
```

## Logging Guidelines

- Use structured logging with `ILogger<T>`
- Include correlation IDs for request tracing
- NEVER log sensitive data (keys, passwords, user content)

```csharp
// ✅ Good
_logger.LogInformation("Album created: {AlbumId} by {UserId}", album.Id, userId);

// ❌ Bad: Logging sensitive data
_logger.LogInformation("User login with password: {Password}", password);
```

## API Response Patterns

```csharp
// Consistent error responses
return Results.Problem(
    title: "Album not found",
    statusCode: 404,
    detail: $"Album with ID {albumId} does not exist"
);

// Consistent success responses
return TypedResults.Ok(new AlbumResponse { Id = album.Id, Name = album.Name });
```
