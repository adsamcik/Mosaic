# apps/android-main — Scoped Instructions

This module is the first real Android Gradle application in Mosaic. Read
`apps/android-main/README.md` for a high-level overview before making changes.

## Tech Stack

- Gradle 8.10.2 + AGP 8.7.3 + Kotlin 2.0.21
- JDK target 17, compileSdk 35, minSdk 26
- AndroidX Activity / AppCompat (no Compose in v1)
- JNA `aar` 5.14.0 (required by generated UniFFI Kotlin)
- Generated UniFFI Kotlin bindings at
  `target/android/kotlin/uniffi/mosaic_uniffi/mosaic_uniffi.kt`,
  copied at build time into `build/generated/source/uniffi/main/kotlin/`.
- Native libraries at `target/android/{arm64-v8a,x86_64}/libmosaic_uniffi.so`,
  copied at build time into `build/generated/jniLibs/{abi}/`.

## Strict Constraints

1. **No INTERNET permission.** This module does not contact a server in v1.
2. **No `READ_MEDIA_*`, `READ_EXTERNAL_STORAGE`, or
   `MANAGE_EXTERNAL_STORAGE`.** Future Photo Picker work uses the system
   Photo Picker with no broad storage permissions.
3. **`android:allowBackup="false"`.** Auto-backup is disabled to avoid
   leaking encrypted app-private state into Google Backup.
4. **abiFilters restricted to `arm64-v8a` + `x86_64`.** No 32-bit builds.
5. **Adapter classes never log handles, key material, file paths, or
   user-identifying material.** Bridges continue the `<redacted>`
   `toString()` pattern from `apps/android-shell`.
6. **`AndroidRust*Api` classes call
   `AndroidRustCoreLibraryLoader.warmUp()` in their `init` blocks** so
   downstream construction order does not matter.
7. **Generated source / native libs are NEVER committed** to git. They live
   under `build/generated/` (ignored). The Rust build is the single source
   of truth; CI runs `scripts/build-rust-android.sh` before Gradle.
8. **No Compose** in v1. Plain `androidx.activity.ComponentActivity` +
   `LinearLayout`/`TextView` only. This keeps the v1 APK <= 6 MB.
9. **Release builds keep minify off in v1.** When release minify is enabled
   later, `proguard-rules.pro` already keeps `uniffi.mosaic_uniffi.**` and
   `com.sun.jna.**`.
10. **Always honor the shell contracts.** Adapter implementations must
    satisfy the `Generated*Api` interfaces from `apps/android-shell` exactly.
    Do not modify shell contracts to fit Rust schema changes; instead extend
    the shell carefully (it is the cross-platform-stable Kotlin contract).

## Patterns

```kotlin
// Adapter class skeleton:
class AndroidRustXxxApi : GeneratedRustXxxApi {
  init {
    AndroidRustCoreLibraryLoader.warmUp()
  }

  override fun someMethod(...): RustXxxFfiResult {
    val uniResult = uniffi.mosaic_uniffi.someGeneratedMethod(...)
    return RustXxxFfiResult(
      code = uniResult.code.toInt(),
      // ... convert UInt/UByte/ULong → Int/Int/Long, etc.
    )
  }
}
```

## Anti-Patterns

- ❌ Writing UI logic inside `bridge/` adapter classes.
- ❌ Logging via `Log.d(TAG, "handle=$handle")` in any adapter.
- ❌ Adding a new permission to the manifest without updating `.instructions.md`.
- ❌ Importing from `target/release/` or any non-generated source path.
- ❌ Using `setProperty("uniffi.component.mosaic_uniffi.libraryOverride", ...)`
  in any code path other than test fixtures.
- ❌ Adding a fallback path that disables JNA / native library loading.

## Testing Requirements

- Every new adapter MUST have a JVM compile-time test in
  `src/test/kotlin/.../bridge/AdapterCompilationContractTest.kt` (extend the
  existing `adapters` list).
- Every adapter that exercises a new Rust function MUST have an instrumented
  smoke test in `src/androidTest/kotlin/.../RustCoreSmokeTest.kt` proving a
  round-trip with synthetic inputs.
- Privacy-redacted `toString()` for any new DTO MUST be covered by a
  contract test in `apps/android-shell/src/test/.../GeneratedRustBridgeContractsTest.kt`.

## Running tests

```powershell
.\scripts\test-android-main.ps1                   # JVM unit tests
.\gradlew.bat :apps:android-main:connectedDebugAndroidTest  # Instrumented (requires emulator)
```
