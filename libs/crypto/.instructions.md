# Crypto Library - Scoped Instructions

> These instructions apply specifically to files in `libs/crypto/`
> 
> ⚠️ **SECURITY CRITICAL**: This library implements zero-knowledge encryption.
> Every line of code must be reviewed with paranoid attention to detail.

## Technology Stack

- **Language:** TypeScript (strict mode)
- **Crypto:** libsodium-wrappers-sumo
- **Testing:** Vitest with 85%+ coverage requirement
- **Build:** TypeScript compiler, outputs to `dist/`

## Cryptographic Primitives (ONLY use these)

| Operation | Primitive | Notes |
|-----------|-----------|-------|
| Symmetric Encryption | XChaCha20-Poly1305 | 24-byte nonce, 32-byte key |
| Asymmetric Encryption | X25519 sealed boxes | For key distribution |
| Signing | Ed25519 | For manifest integrity |
| Key Derivation | Argon2id | Password → Master Key |
| Key Expansion | HKDF-SHA256 | Master Key → Root Key |

## Strict Invariants

### Nonce Handling (CRITICAL)
```typescript
// ✅ ALWAYS: Generate fresh 24-byte nonce per encryption
const nonce = sodium.randombytes_buf(24);

// ❌ NEVER: Reuse nonces, use counters, or derive nonces
const nonce = previousNonce;  // CATASTROPHIC
```

### Memory Safety
```typescript
// ✅ ALWAYS: Zero sensitive buffers after use
function deriveKey(password: string): Uint8Array {
  const masterKey = sodium.crypto_pwhash(...);
  try {
    const derivedKey = hkdf(masterKey, ...);
    return derivedKey;
  } finally {
    sodium.memzero(masterKey);  // REQUIRED
  }
}

// ❌ NEVER: Let keys go out of scope without zeroing
function deriveKey(password: string): Uint8Array {
  const masterKey = sodium.crypto_pwhash(...);
  return hkdf(masterKey, ...);  // masterKey leaked!
}
```

### Input Validation
```typescript
// ✅ ALWAYS: Validate before crypto operations
export function encryptShard(data: Uint8Array, key: Uint8Array): EncryptedShard {
  if (key.length !== 32) {
    throw new Error('Invalid key length: expected 32 bytes');
  }
  if (data.length === 0) {
    throw new Error('Cannot encrypt empty data');
  }
  // ... proceed with encryption
}

// ❌ NEVER: Trust inputs blindly
export function encryptShard(data: Uint8Array, key: Uint8Array): EncryptedShard {
  const nonce = sodium.randombytes_buf(24);
  return sodium.crypto_aead_xchacha20poly1305_ietf_encrypt(data, null, null, nonce, key);
}
```

## Key Hierarchy Reference

```
L0 (Master)  = Argon2id(password, salt)     # NEVER stored
L1 (Root)    = HKDF(L0, account_salt)       # NEVER stored  
L2 (Account) = random(32), wrapped by L1   # Stored encrypted
L3 (Epoch)   = ReadKey + SignKey per album # Distributed to members
```

## Envelope Format (64 bytes header)

```
Offset | Size | Field     | Description
-------|------|-----------|----------------------------------
0      | 4    | Magic     | 0x4D4F5341 ("MOSA")
4      | 1    | Version   | Currently 0x01
5      | 4    | Epoch     | Epoch ID (big-endian uint32)
9      | 4    | Shard     | Shard index (big-endian uint32)
13     | 24   | Nonce     | XChaCha20 nonce
37     | 27   | Reserved  | MUST be zero, validated on decrypt
```

- **AAD** = entire 64-byte header (provides authenticated encryption)
- Reserved bytes MUST be validated as zero on decrypt

### Error Types
```typescript
// Define specific error classes
export class CryptoError extends Error {
  constructor(message: string, public readonly code: string) {
    super(message);
    this.name = 'CryptoError';
  }
}

export class DecryptionError extends CryptoError {
  constructor(message: string) {
    super(message, 'DECRYPTION_FAILED');
  }
}

export class SignatureVerificationError extends CryptoError {
  constructor(message: string) {
    super(message, 'SIGNATURE_INVALID');
  }
}
```

## Forbidden Patterns

- ❌ Implementing custom crypto algorithms
- ❌ Using `Math.random()` for any cryptographic purpose
- ❌ Storing keys in plain text (localStorage, cookies, etc.)
- ❌ Comparing keys with `===` (use `sodium.memcmp`)
- ❌ Using CBC mode or any non-AEAD cipher
- ❌ Deriving nonces from data (use `randombytes_buf`)
